# 版权与风格边界

## 使用方式

在分析独立游戏美术方向、写 mood board、生成提示词或制定外包美术规范前，先用本文件划定边界。以下不是法律意见；若项目将商业发行或争议风险较高，应让权利顾问审查。我的判断依据：默认不能处理未授权游戏资源；若用户明确授权本机目录，可做只读学习分析，但输出仍要转译为原创设计原则。

## 可做、谨慎、禁止

| 类型 | 处理方式 |
|---|---|
| 观察画面中的抽象特征 | 可做。记录构图、调色、明暗层级、形状语言、动效节奏、UI 信息密度等。 |
| 引用公开页面或用户提供截图 | 可做。标注来源、日期和用途；不要把截图当作可复用资产。 |
| 受某作品启发的原创方向 | 可做。至少改变主题、角色设定、图标、关卡结构、镜头语言、色彩比例和资产形态中的多个关键维度。 |
| “像某游戏一样”的需求 | 谨慎。改写为可执行的独立描述，例如“高对比剪影、低色数、短帧数攻击预备动作”。 |
| 用户明确授权的本地游戏目录只读导出、索引或 IL2CPP/native 行为映射 | 谨慎。只写入学习目录，只总结抽象规律、结构、方法名、RVA/VA 和行为事实；不要分发原始素材或反编译代码。 |
| 使用原作角色、logo、UI、图标、tileset、sprite sheet、音效、字体包 | 禁止，除非用户提供明确授权。 |
| 未经授权下载、解包、提取、反编译、复用商业游戏资源 | 禁止。不要提供工具、步骤或替代下载路径。 |
| 生成几乎可混淆的角色、怪物、界面或宣传图 | 禁止。改为生成不同主题、不同轮廓、不同配色和不同符号系统。 |

## 风格转译方法

1. 把参考作品拆成抽象变量：色相范围、饱和度、对比度、像素密度、镜头距离、形状比例、边缘处理、材质暗示、动画节拍。
2. 删除不可复制元素：角色名称、logo、独有图标、UI 布局、关卡图案、具体 sprite、专有怪物轮廓、宣传图构图。
3. 为原创项目建立新约束：世界观、目标平台、制作人力、交互优先级、可读性标准、资产尺寸、调色板规则。
4. 生成“相似体验目标”，而不是“相似表面结果”。例如把“像 A 游戏”改成“在小屏幕上保持高可读性的暖色前景与冷色背景分离”。
5. 记录每个参考样本的来源和边界：哪些特征可学习，哪些元素不可复用。

## 高风险措辞替换

| 高风险表达 | 建议改写 |
|---|---|
| 复制这个游戏的 UI | 提取信息层级原则，重新设计 UI 结构、图标和视觉语言。 |
| 做成和某角色一样 | 保留功能定位，重设轮廓、服装、色彩、道具和叙事身份。 |
| 直接用原图当素材 | 仅把原图作为研究样本，重新绘制原创资产。 |
| 解包看看里面怎么做 | 仅在用户明确授权的本地目录中只读学习；否则只分析公开截图、商店页、官方文档或用户自有工程。 |
| 完全照着这个 tileset 做 | 定义新的网格规格、主题纹理、边缘规则和装饰语汇。 |

## 待验证

- 用户提供的本地路径是否确属授权学习范围。
- 目标项目是否有额外 EULA、商标或发行限制。
- 最终报告是否只保留抽象原则、原创约束和可核验来源。

## 参考来源

- U.S. Copyright Office, [What Does Copyright Protect?](https://copyright.gov/help/faq/faq-protect.html)：版权保护表达，不保护想法本身。
- U.S. Copyright Office, [Circular 31: Ideas, Methods, or Systems](https://www.copyright.gov/circs/circ31.pdf)：思想、方法、系统不受版权保护，具体文字或图像表达可能受保护。
- USPTO, [What is a trademark?](https://www.uspto.gov/trademarks/basics/what-trademark)：商标可包括识别商品或服务来源的文字、短语、符号或设计。
- 用户任务：明确要求脚本不得下载或提取任何游戏资源。
