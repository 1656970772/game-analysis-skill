# 本地导出与预览需求

## 使用方式

当用户要求“全量本地导出、全量图片画廊、覆盖层预览、EXE/Electron 本地壳、右键打开素材目录”时，按本文件写规格或实现计划。边界：只处理用户授权的本地路径；未授权资源不扫描、不导出、不上传；授权后的原始导出只留在本机学习目录；不修改游戏安装目录。

## 功能规格

| 功能 | 要求 | 边界 |
|---|---|---|
| 全量本地导出 | 对授权路径生成完整本地导出、索引、统计、调色板摘要、缩略图路径、原创分析报告，可选 JSON/Markdown/HTML；Unity 项目先生成 AssetRipper `ExportedProject`，再生成 UnityPy 可读对象索引 | 原始导出只在本机学习目录使用；对外交付只给索引、缩略图和原创总结 |
| 关键角色动画帧 | 根据 AnimatorController 与 `.anim` 的 `m_Sprite` 引用，先从 AssetRipper 工程切片，再用 UnityPy 对照可读原始 bundle，按 `attack01_1.png`、`idle_1.png` 这类规则整理 | 仅用于学习动作拆分和节奏；原创项目必须重新绘制角色与动作表达 |
| 全量图片画廊 | 展示授权目录中的图片、尺寸、格式、主色、来源路径、标签和筛选 | 不展示未授权内容；默认只读 |
| 当前页覆盖层预览 | 点击卡片在当前页面叠加预览层，再次点击同一素材、点背景、点图片或按 Esc 关闭；可扩展网格、取色、明度、构图辅助线、注释图层 | 覆盖层不改原图；注释另存到工具工作区 |
| EXE/Electron 本地壳 | 打包为离线桌面工具，保留浏览、筛选、导出、报告生成 | 默认禁用外网；敏感路径不写入日志 |
| 右键打开素材目录 | 在 EXE/Electron 壳内右键素材卡片或预览图，直接用系统文件管理器定位素材所在目录或选中素材 | 仅调用系统文件管理器打开本地目录，不执行脚本或提升权限 |

## 推荐实现约束

- 工作区与项目输出目录分离：缓存、缩略图、报告写入用户指定的工具工作区。
- 所有写入动作先显示目标路径；批量导出前生成清单。
- 对第三方参考样本保留来源链接和观察记录，不生成可复用资产包。
- 图片预览优先生成尺寸受限的缩略图；原图仅在用户授权目录内按需查看。
- Electron/EXE 壳保留本地模式开关：离线、只读、允许导出、允许打开目录。
- 关键角色动画输出必须包含逐帧 CSV、动画级 CSV、manifest 和命名规则说明。
- 我的判断：覆盖层除了查看大图，还应支持记录色彩、形状语言、角色轮廓、UI/HUD 可读性和制作约束，便于把观察转译成原创项目规范。

## 待验证

- EXE 打包后在中文路径、空格路径和移动位置后仍能找到画廊与授权素材目录。
- 右键定位只打开系统文件管理器，不执行脚本、不联网、不提升权限。
- 覆盖层关闭、翻页和筛选后不残留高亮状态或遮挡界面。

## 验证清单

- 授权：每个扫描路径都有用户授权说明。
- 只读：扫描、画廊、覆盖层不会修改原项目文件。
- 导出：导出目录不是项目输出目录，且包含导出清单。
- 画廊：能全量列出图片，缺失/损坏图片有错误提示。
- 覆盖层：打开、关闭、缩放、翻页后不遮挡核心内容。
- 右键定位：只打开素材所在目录或选中文件，不执行脚本、不联网、不提升权限。
- 本地壳：离线可运行；打包后路径、权限、中文文件名正常。
- 边界：报告中已标注来源、权限状态、不可复制元素和待验证项。
