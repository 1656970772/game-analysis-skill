# 场景关卡搭建复盘模板

## 资深角色视角

- 关卡设计师：先复盘空间目标、路线、节奏、可见性和可交互物。
- 环境美术/主美：检查 art pass 是否保留 blockout 功能，并用光照、色彩、地标引导玩家。
- 技术美术：检查 Prefab/Actor/Tile/Room 复用、碰撞、导航和性能成本。
- 我的判断：场景复盘必须写清授权边界、制作约束、调色板/形状语言如何服务可读性。

## 引擎分支

| 引擎/工具 | 场景证据 |
|---|---|
| Unity | Scene、Prefab、Tilemap、SpriteRenderer、Collider、NavMesh、Lighting |
| Unreal | Level/World、Actor、Component、Landscape、Spline、Lighting、Blueprint trigger |
| GameMaker | Room、layer、tilemap、instance、object、collision mask |
| Godot | Scene tree、TileMap、Node2D/3D、CollisionShape、Navigation、signals |
| 未知引擎 | 地图文件、房间表、坐标数据、资源命名、碰撞/导航文本 |

## 必填分析细节大纲

1. 关卡目标：教学、战斗、探索、叙事、节奏转换。
2. 空间拓扑：主路、支路、回环、门槛、捷径、锁钥。
3. Blockout/灰盒：尺寸、比例、视线、动线、遮挡、地标。
4. 交互提示：可互动/危险/奖励/不可达区域如何被视觉标记。
5. Art pass：主题、材质、光照、色彩、FX、音频如何保留玩法功能。
6. 资源复用：Prefab/Actor/Tile/Room 模块、变体、命名、组合方式。
7. 节奏：遭遇密度、安全区、奖励点、难度变化。
8. 性能：遮挡、批次、对象数量、粒子、灯光、贴图尺寸。
