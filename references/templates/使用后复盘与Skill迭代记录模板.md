# 使用后复盘与Skill迭代记录模板

## 资深角色视角

- 工具负责人：把一次项目经验沉淀成可复用流程、脚本、测试和工具清单。
- 架构师：只沉淀通用知识，不把商业项目私有路径、素材、密钥或可复用源码片段写入 skill。
- QA：用失败场景补测试，防止后续 agent 重复踩坑。
- 我的判断：复盘必须记录授权边界、原创化、调色板、形状语言、制作约束和待验证项是否需要补模板。

## 引擎分支

| 引擎/工具 | 复盘重点 |
|---|---|
| Unity | Unity 本地工具链、IL2CPP/Animator/Scene 的新失败模式和补充脚本 |
| Unreal | Unreal 包体格式、只读工具链、chunk/mount point、加密/版本阻塞 |
| GameMaker | UTMT、YYC、texture page、sprite frame、audio group、路径编码 |
| Godot | pck、scene/resource、signals、autoload、导出包工具 |
| 未知引擎 | 新格式特征、可信工具、下载隔离、小样本验证、reference 草案 |

## 必填分析细节大纲

1. 本次任务：输入路径、输出目录、模式、命中引擎。
2. 成功项：哪些流程可复用。
3. 失败项：工具、格式、权限、GUI、路径、性能、磁盘空间。
4. Skill 缺口：文档、脚本、工具、测试、模板、合规规则。
5. 本次已更新：文件路径、测试、验证命令。
6. 未完成待办：明确下一步、原因、触发条件。
7. 禁止沉淀内容：私有路径、原始素材、密钥、可复用源码片段。
