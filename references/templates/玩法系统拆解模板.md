# 玩法系统拆解模板

## 资深角色视角

- 游戏设计师：用机制、动态、体验拆解核心循环和玩家决策。
- 系统策划：关注资源、风险、胜负条件、难度曲线和系统耦合。
- UX/主程：把输入、状态和反馈链路落到可验证证据。
- 我的判断：玩法模板也要记录系统如何影响调色板、形状语言、UI 提示和原创化制作约束。

## 引擎分支

| 引擎/工具 | 玩法证据入口 |
|---|---|
| Unity | Scene/Prefab、Animator、Input、ScriptableObject、TextAsset/DataTable |
| Unreal | Actor/Component、Blueprint、DataAsset、Gameplay tag、Level/World partition |
| GameMaker | Object/Event、Room、GML script、sprite animation、alarm/timeline |
| Godot | Node/Scene、Signal、Autoload、Resource、GDScript |
| 未知引擎 | 配置表、文本、资源命名、日志、可执行字符串、关卡文件 |

## 必填分析细节大纲

1. 核心循环：30 秒、3 分钟、长期成长。
2. 输入-状态-反馈链路：玩家输入、系统解释、视觉/音频/数值反馈。
3. 目标与失败：胜利、失败、惩罚、重试、恢复。
4. 资源系统：生命、能量、货币、冷却、风险资源。
5. 决策类型：空间、时机、资源、构筑、路线、风险。
6. 随机性与信息透明度：随机发生在哪里，玩家是否能预判。
7. 难度曲线：敌人、关卡、资源、惩罚、教学节奏。
8. 系统耦合：战斗、移动、关卡、UI、美术、音频如何互相支撑。
9. 可原创化原则：抽象规则，不复制数值表、关卡和 UI 图标。
