# 动画与手感拆解模板

## 资深角色视角

- 动画负责人：检查动作是否清楚表达意图，同时不牺牲输入响应。
- Gameplay 程序：检查状态机、取消窗口、输入缓冲、hit stop、镜头和反馈触发。
- 技术美术/VFX：检查残影、粒子、音效、屏幕震动和视觉层级是否服务手感。
- 合规边界：动作分析只总结原创化节奏原则；必要时记录调色板、形状语言、可读性和制作约束。

## 引擎分支

| 引擎/工具 | 动画证据 |
|---|---|
| Unity | AnimatorController、AnimationClip、Sprite slice、Timeline、ParticleSystem、Cinemachine |
| Unreal | Animation Blueprint、Montage、Notify、Niagara、Camera shake |
| GameMaker | sprite frames、image_speed、alarm/timeline、object event、room speed |
| Godot | AnimationPlayer、AnimationTree、AnimatedSprite2D/3D、Tween、Signal |
| 未知引擎 | 序列帧命名、动作表、状态字符串、粒子贴图、音效触发名 |

## 必填分析细节大纲

1. 基础动作：idle、run、jump、attack、hit、death、interact。
2. 帧数和节奏：预备、释放、命中、恢复、循环、过渡。
3. 输入响应：输入缓冲、取消窗口、转向、锁定、自动修正。
4. 反馈增强：hit stop、屏幕震动、残影、粒子、音效、控制器震动。
5. 状态机：状态、条件、参数、事件、动画通知。
6. 手感三栏：physicality、amplification、support。
7. 可读性：敌我动作差异、危险预告、攻击范围、UI/音频提示。
8. 原创化规则：保留抽象节奏，不复制动作帧和角色设计。

## 引用来源

- [Designing Game Feel: A Survey](https://arxiv.org/abs/2011.09201)：用于 physicality、amplification、support 等手感分析维度。
- [Game Feel - Steve Swink](https://books.google.com/books?id=3oFjDwAAQBAJ)：用于输入、反馈和虚拟感觉的设计框架。
- [The Art of First Person Animation for Destiny](https://sandbox.gdcvault.com/play/1022298/The-Art-of-First-Person)：用于持续可见动作、响应性和物理感的动画分析。
- [GameMaker Objects and Instances](https://manual.gamemaker.io/lts/en/Quick_Start_Guide/Objects_And_Instances.htm)：用于 GameMaker event 与每帧更新关系。
