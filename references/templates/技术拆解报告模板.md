# 技术拆解报告模板

## 资深角色视角

- 主程/架构师：把引擎、资源、渲染、输入、音频、存档、数据和性能拆成可验证子系统。
- 技术美术：关注资源依赖、材质/Shader、VFX、动画系统和运行时成本。
- 构建工程师：关注包体、chunk、AssetBundle/Addressables、导出工具和可复现命令。
- 合规边界：技术结论也要服务原创化学习；涉及调色板、形状语言和资产制作约束时只写抽象规则。

## 引擎分支

| 引擎/工具 | 技术拆解重点 |
|---|---|
| Unity | Scene/Prefab、Animator、ScriptableObject、Addressables/AssetBundle、Profiler CPU/GPU/Rendering/Memory/Audio |
| Unreal | Actor/Component、Blueprint/C++、Pak/Chunk/Cook、AssetRegistry、Unreal Insights CPU/GPU/Load/Memory |
| GameMaker | Object/Event、Room、Texture Page、Audio Group、GML/YYC、Debugger/Profiler |
| Godot | Node/Scene、Resource、Signal、Autoload、`.tscn/.tres/.gd`、Profiler |
| 未知引擎 | 包格式、资源加载入口、配置文本、可执行导入表、日志、工具验证结果 |

## 必填分析细节大纲

1. 引擎与版本证据：文件结构、manifest、日志、可执行文件、包体格式。
2. 构建与包体：资源包、chunk、AssetBundle、pak、texture page、scene/resource。
3. 渲染路径：管线、材质、Shader、后处理、光照、批处理、LOD、粒子。
4. 资源加载：同步/异步、引用表、外部载荷、StreamingAssets、DataTable/配置。
5. 输入与控制：输入层、动作映射、缓冲、状态机入口。
6. 动画系统：Animator/Flipbook/StateMachine/Timeline/Spine/自研。
7. UI 与本地化：UI atlas、字体、文本、状态反馈、语言表。
8. 音频：AudioClip/sound/music、mixer、触发关系、音频组。
9. 存档与配置：存档路径、格式、用户设置、难度/数值配置。
10. 性能证据：Profiler/Insights/Debugger、采样条件、瓶颈分类、待验证项。

## 引用来源

- [Unity Profiler overview](https://docs.unity3d.com/Manual/Profiler.html)：用于 Unity 性能、渲染、音频、内存等模块化检查。
- [Unity Addressables Diagnostic Tools](https://docs.unity3d.com/Packages/com.unity.addressables@latest/manual/DiagnosticTools.html)：用于资源依赖和 Addressables 构建报告检查。
- [Unreal Insights](https://dev.epicgames.com/documentation/en-us/unreal-engine/unreal-insights-in-unreal-engine)：用于 Unreal trace、CPU/GPU、加载和内存分析。
- [Unreal Cooking Content and Creating Chunks](https://dev.epicgames.com/documentation/en-us/unreal-engine/cooking-content-and-creating-chunks-in-unreal-engine)：用于 Unreal cook/chunk/pak 结构分析。
- [GameMaker Profiler](https://manual.gamemaker.io/monthly/en/IDE_Tools/The_Debugger/The_Profiler.htm)：用于 GameMaker 事件、脚本和函数热点记录。
