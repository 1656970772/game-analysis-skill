# 美术规范与风格报告模板

## 资深角色视角

- 主美/概念设计师：把参考项目转译为色彩、形状、材质、剪影、UI 语法和生产规则。
- 技术美术：检查资产规格、图集、材质、Shader、VFX、字体和引擎导入约束。
- 关卡美术：确保风格服务可读性、动线和交互提示。

## 引擎分支

| 引擎/工具 | 美术规范证据 |
|---|---|
| Unity | Texture2D/Sprite、Material/Shader、Prefab、Animator、SpriteAtlas、TextMeshPro |
| Unreal | Texture/Material、Niagara/Cascade、Widget、DataAsset、Level assets |
| GameMaker | texture page、sprite、background、font、object draw event |
| Godot | Texture、Material、Theme、Control、TileSet、AnimationPlayer |
| 未知引擎 | 图片尺寸、命名、目录结构、材质文本、shader 字符串、UI atlas |

## 必填分析细节大纲

1. 风格定位：题材、时代、情绪、玩家视角。
2. 参考矩阵：观察事实、来源、不可复制元素、可迁移原则。
3. 色彩：主色、辅助色、强调色、明度层级、危险/奖励/路径色。
4. 形状与剪影：玩家、敌人、NPC、道具、交互物、背景。
5. 材质语言：金属、木、石、布、魔法/科技、磨损与边缘。
6. UI/HUD：字号、图标、边框、状态反馈、可访问性。
7. 资产规格：像素密度、tile、角色高度、描边、缩放、锚点。
8. 生产规则：命名、分层、atlas、LOD、复用、交付格式。
9. 原创化建议：换题材、换轮廓、换色彩关系、换 UI 语法。
