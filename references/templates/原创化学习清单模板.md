# 原创化学习清单模板

## 资深角色视角

- 创意总监：把参考项目拆成可迁移原则，而不是做相似外观、关卡或代码。
- 主美/概念设计师：将风格参考转译为新题材、新剪影、新色彩和新 UI 语法。
- 主程/玩法设计师：把系统结构改写成原创原型练习和验证问题。
- 我的判断：原创化清单必须包含调色板、形状语言、制作约束和待验证风险。

## 引擎分支

| 引擎/工具 | 原创化练习方向 |
|---|---|
| Unity | 用不同数据结构重做输入、Animator、ScriptableObject、Prefab 组合 |
| Unreal | 用不同 Actor/Component、Blueprint/DataAsset、Level 结构重做原型 |
| GameMaker | 用不同 object/event/room/sprite 组织重做 2D 原型 |
| Godot | 用不同 node/scene/signal/resource 结构重做原型 |
| 未知引擎 | 只迁移观察到的抽象原则，不迁移私有格式、素材和不可复用代码逻辑 |

## 必填分析细节大纲

1. 可学习原则：技术、玩法、美术、关卡、动画、管线。
2. 不可复制内容：角色、logo、UI、图标、tileset、关卡布局、源码、原始素材。
3. 原创替代表：换题材、换剪影、换色彩、换规则、换数值、换反馈。
4. 小型原型：1 日、3 日、1 周练习任务。
5. 验收问题：是否可玩、可读、不同于参考、可量产。
6. 风险审查：是否过度相似、是否包含第三方素材、是否泄露导出内容。
