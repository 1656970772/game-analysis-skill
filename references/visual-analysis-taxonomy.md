# 视觉分析分类法

## 使用方式

用本分类法把游戏画面拆成可讨论、可执行、可原创化的设计变量。分类本身属于我的方法论判断，依据是 GameAnalysis Skill 需要支持风格研究、资产盘点、调色板分析和交付模板。

## 分析维度

| 维度 | 观察内容 | 输出问题 |
|---|---|---|
| 类型与视角 | 2D/3D、俯视/横版/等距/第一人称、镜头距离、可见交互范围 | 玩家需要在多远距离读懂角色、敌人和可交互物？ |
| 像素与分辨率 | 基准 tile 尺寸、角色高度、描边粗细、像素密度、缩放策略 | 最小可读单位是多少？哪些资产必须共用网格？ |
| 构图 | 前中后景、留白、焦点、屏幕边缘信息、战斗可读区 | 画面如何把玩家视线带到目标？ |
| 形状语言 | 圆/方/尖比例、剪影复杂度、重复图案、敌我轮廓区分 | 不看颜色时是否还能识别功能？ |
| 线条与边缘 | 有无描边、内线、抗锯齿、破边、噪点、边缘高光 | 边缘处理如何服务可读性和手感？ |
| 色彩 | 主色、辅色、强调色、冷暖关系、饱和度、色数限制 | 颜色如何区分阵营、危险、奖励和路径？ |
| 明度 | 亮暗层级、背景压暗、前景提亮、交互物高亮 | 灰度图里核心信息是否仍清楚？ |
| 光照与材质 | 光源方向、阴影硬度、金属/木/石/布的表现方式 | 材质是否用最少像素表达清楚？ |
| 动效 | 帧数、预备/释放/恢复、残影、屏幕震动、粒子密度 | 动画是否先服务交互反馈，再服务装饰？ |
| UI/HUD | 字号、图标语法、边框、提示层级、色盲可读性 | UI 是否与场景争抢注意力？ |
| 世界叙事 | 道具磨损、环境标识、生态逻辑、时代线索 | 资产是否暗示玩法和故事，而不是只做装饰？ |
| 制作约束 | 团队规模、工具链、批量生产方法、复用规则 | 哪些规则能减少后续资产成本？ |

## 研究矩阵

| 样本 | 观察事实 | 我的判断 | 可迁移原则 | 原创化方向 |
|---|---|---|---|---|
| 样本名 | 来自截图或来源的可核验事实 | 基于事实的解读 | 可应用到新项目的抽象规则 | 改主题、改轮廓、改配色、改 UI 语法 |

## 独立游戏常用约束

- 小团队项目优先定义可复用规则：tile 边缘、角色高度、轮廓复杂度、调色板、UI 组件、动效帧数。
- 低预算美术方向应先保证识别、反馈和量产，再追求装饰复杂度。
- 像素项目要同时规定源分辨率和显示缩放，避免不同资产出现不一致的像素密度。
- 视觉研究报告应输出“能画出来的规则”，不是只输出情绪词。

## 快速观察清单

1. 用一句话描述体验目标。
2. 截取或引用 3 到 8 个授权样本，并记录来源。
3. 对每个样本填写“事实、判断、可迁移原则、不可复制元素”。
4. 汇总成原创方向：色彩规则、形状规则、资产尺寸、动效节奏、UI 语法、制作限制。
5. 用 `guideline_lint.py` 检查最终文字是否暗示复制或资源提取。
